    1. Create `internal/database/database.go` per RESEARCH.md Pattern 2:
       ```go
       package database

       func Connect(ctx context.Context, dsn string) (*pgxpool.Pool, error) {
           pool, err := pgxpool.New(ctx, dsn)
           if err != nil {
               return nil, fmt.Errorf("pgxpool.New: %w", err)
           }
           if err := pool.Ping(ctx); err != nil {
               pool.Close()
               return nil, fmt.Errorf("database ping failed: %w", err)
           }
           return pool, nil
       }
       ```

    2. Create `db/migrations/00001_create_posts.sql` with goose-format migration:
       ```sql
       -- +goose Up
       CREATE TABLE posts (
           id          BIGSERIAL PRIMARY KEY,
           title       TEXT NOT NULL,
           slug        TEXT NOT NULL UNIQUE,
           body        TEXT NOT NULL DEFAULT '',
           rendered_html TEXT NOT NULL DEFAULT '',
           published   BOOLEAN NOT NULL DEFAULT false,
           created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
           updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
           deleted_at  TIMESTAMPTZ
       );

       CREATE INDEX idx_posts_slug ON posts (slug);
       CREATE INDEX idx_posts_published ON posts (published) WHERE deleted_at IS NULL;

       -- +goose Down
       DROP TABLE IF EXISTS posts;
       ```

    3. Create `internal/database/migrations.go` per RESEARCH.md Pattern 3:
       - Use `//go:embed` to embed `db/migrations/*.sql` -- note the embed path. Since the file lives in `internal/database/`, the relative path to `db/migrations/` is `../../db/migrations/*.sql`.
       - For goose compatibility with pgxpool, use `pgx/v5/stdlib` to get a `*sql.DB` from the pool: `stdlib.OpenDBFromPool(pool)`. This satisfies goose's `*sql.DB` requirement while keeping the main app on native pgx.
       - Call `goose.SetBaseFS(migrationsFS)`, `goose.SetDialect("postgres")`, then `goose.Up(sqlDB, "db/migrations")`.
       - The path argument to `goose.Up` must match the directory structure inside the embed FS. Since the embed directive is `//go:embed ../../db/migrations/*.sql`, the FS root contains `db/migrations/`, so the path is `"db/migrations"`.
