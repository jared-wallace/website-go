## Page Layouts

### Login page (`/admin/login`)

- Centered vertically and horizontally in viewport
- Width: 400px max, full-width on mobile
- Contains: site name (`.site-name`), nautical tagline, form with email + password fields, "Sign In" submit button (`.cta-button` style), error flash below button
- No nav bar (unauthenticated state — per D-03)
- No footer

### Dashboard (`/admin/posts`)

- Full `.site-nav` with admin badge replacing public tagline
- Filter tab row immediately below nav
- Post table: title (truncated at 60 chars), status pill, published date, action links
- **Focal point:** The post title column is the primary visual anchor — rendered in `var(--font-display)` at 18px/700, full-width, drawing the eye before the status pill or date. Status pill and action links are secondary and visually recede (14px label size, muted or surface background).
- Empty state visible when filtered tab has zero results
- Footer omitted (functional tool, not a reading experience)

### Editor (`/admin/posts/new`, `/admin/posts/{id}/edit`)

- Nav present (same admin nav)
- Title field + slug field above the split pane
- Split pane: editor left, preview right (50/50) on desktop
- Mobile: stacked, tab toggle bar visible
- Toolbar below split pane: "Save Draft" button + "Publish" button + status indicator
- Keyboard shortcut: Ctrl+S / Cmd+S triggers Save Draft (same as "Save Draft" button)
- Debounce: 300ms after keyup before preview fetch fires (midpoint of the 200-500ms discretion range)

## Interaction Contracts

### Split-pane preview

- Debounce: 300ms after last keyup event
- Request: `POST /admin/preview` with form body `markdown=<encoded content>`
- Response: HTML fragment — inject directly into `.admin-preview-pane` inner content
- Loading state: preview pane shows current content until response arrives (no spinner — latency is sub-100ms on same server)
- Error state: preview pane shows "Preview unavailable — check your connection" in muted text

### Slug auto-generation

- Triggers on title field input event
- Algorithm: lowercase → replace non-alphanumeric runs with `-` → strip leading/trailing `-`
- Manual override: admin can click slug field and type directly; auto-generation stops once manually edited
- Indicator: slug field shows generated value in real time

### Delete confirmation

- "Delete" link click opens native `<dialog>` element (no JS modal library)
- Dialog copy: "Delete this post? It will be moved to the Deleted tab and can be restored." with "Keep Post" and "Confirm Delete" buttons
- "Keep Post" closes dialog, no action taken
- "Confirm Delete" submits `POST /admin/posts/{id}/delete` form inside the dialog

### Publish/Unpublish

- Acts immediately on click — no confirmation (easily reversible per D-15)
- Page reloads to dashboard after action (standard form POST → redirect pattern)
- Flash success message appears at top of dashboard on redirect

### Save Draft / Publish (editor)

- Both buttons submit the same form; hidden input distinguishes action (`action=draft` vs `action=publish`)
- After save: redirect to `/admin/posts/{id}/edit` with flash success
- Validation error: redirect back to editor with flash error and preserved form values

## Accessibility Notes

- All form inputs have associated `<label>` elements (explicit `for`/`id` pairing)
- Error messages use `role="alert"` so screen readers announce them on injection
- `<dialog>` element uses native focus trap behavior; no JS focus management required
- Touch targets: all interactive elements maintain `min-height: 44px; min-width: 44px` per existing site pattern
- Focus-visible: inherited from `:focus-visible { outline: 2px solid var(--color-accent); outline-offset: 2px }` in main.css
- Split-pane editor textarea: `aria-label="Markdown editor"`, preview pane: `aria-label="Live preview"` + `aria-live="polite"`
