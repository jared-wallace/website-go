# Phase 7 — UI Design Contract: Rebrand + Navigation

> Visual and interaction contract for Phase 7. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase is a surgical rebrand: string replacements across templates, RSS handler, and meta tags,
> plus one new element (RSS icon in footer). No new layout, no new pages, no new components.
> All tokens below are extracted from the live codebase — nothing is speculative.

## RSS Icon — New Element Specification

This is the only net-new visual element in Phase 7.

| Property | Value |
|----------|-------|
| Location | `web/templates/base.html` footer, right of the copyright `<p>` tag |
| Markup pattern | Inline SVG, `aria-label="RSS feed"`, `aria-hidden="true"` on the SVG itself |
| Icon shape | Classic RSS broadcast icon: circle + two arcs (Feather-style, `stroke-width="2"`, no fill) |
| Size | 16px × 16px (`width="16" height="16"`) — matches footer font size context |
| Link target | `href="/rss"` |
| Link element | `<a>` wrapping the SVG, `class="rss-link"` |
| Touch target | `min-width: 44px; min-height: 44px; display: inline-flex; align-items: center; justify-content: center` — matches `.dark-toggle` pattern |
| Color at rest | `var(--color-text-muted)` |
| Color on hover | `var(--color-accent)` |
| Color transition | `color 200ms ease` — matches existing card/toggle transition timing |
| Margin | `margin-left: 8px` from copyright text |
| Implementation | Inline SVG in `base.html`, CSS in `main.css` under `/* --- Footer --- */` section |

No new CSS class needed for the `<a>` wrapper beyond `.rss-link`. The SVG `currentColor` inherits from the anchor's color property.

Source: `CONTEXT.md` D-01, D-02; dark-toggle SVG pattern in `base.html` lines 30–44.

## Copywriting Contract

All copy changes are string replacements. New strings listed below are the target values.

| Element | Current (to remove) | Target (new value) |
|---------|--------------------|--------------------|
| Browser title default | `The Log` | `The Wild Meridian` |
| Nav `.site-name` text | `The Log` | `The Wild Meridian` |
| Footer copyright | `© {{.Year}} The Log` | `© {{.Year}} Jared Wallace` |
| RSS alternate link title | `The Log` | `The Wild Meridian` |
| OG `og:title` default | `The Log` | `The Wild Meridian` |
| Twitter `twitter:title` default | `The Log` | `The Wild Meridian` |
| Admin nav `.site-name` | `The Log -- Back Office` | `The Wild Meridian -- Back Office` |
| Admin login `<h1>` | `The Log` | `The Wild Meridian` |
| RSS channel `<title>` | `The Log` | `The Wild Meridian` |
| RSS test assertion | `<title>The Log</title>` | `<title>The Wild Meridian</title>` |
| Handler test assertion | `og:title" content="The Log"` | `og:title" content="The Wild Meridian"` |

Strings that do NOT change:
- Tagline: `dispatches from the deep end` — stays as-is (out of scope per `REQUIREMENTS.md`)
- Admin login subtitle: `Back Office` — the admin-login.html `<p>` element stays unchanged
- RSS channel description: `dispatches from the deep end` — stays as-is

Primary CTA: None — this phase has no user-facing calls to action. The only new interactive element is the RSS icon link (no CTA label needed, icon is self-explanatory with `aria-label="RSS feed"`).

Empty state: Not applicable — no new empty states introduced.

Error state: Not applicable — no new error paths introduced.

Destructive actions: None in Phase 7.

Source: `CONTEXT.md` decisions D-03, D-04; canonical refs list; `REQUIREMENTS.md` Out of Scope section.
