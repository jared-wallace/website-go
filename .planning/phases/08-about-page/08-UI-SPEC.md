## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline code padding, tag gaps |
| sm | 8px | Nav padding vertical, element spacing |
| md | 16px | Body paragraph margin, section rhythm |
| lg | 24px | Nav horizontal padding, footer padding, card padding |
| xl | 32px | Post body heading margins, error page padding |
| 2xl | 48px | Empty state vertical padding, pagination top margin |
| 3xl | 64px | Container vertical padding |

Exceptions:
- Dark mode toggle touch target: minimum 44px × 44px (already enforced via `.dark-toggle` in existing CSS)
- About nav link: minimum 44px touch target height on mobile (same pattern as pagination links)

**Source:** Measured from `.container`, `.site-nav`, `.site-footer`, `.post-body` rules in `web/static/main.css`.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 18px | 400 (regular) | 1.7 | Lora, Georgia, serif |
| Label / Meta | 14px | 400 (regular) | 1.4 | Lora, Georgia, serif |
| Heading (h2 in post body) | 28px | 700 (bold) | 1.2 | Playfair Display, Times New Roman, serif |
| Display (page title, site name) | 36px | 700 (bold) | 1.15 | Playfair Display, Times New Roman, serif |

**About page heading rules:**
- Page-level `<h1>` ("About") rendered at 36px / weight 700 / Playfair Display / line-height 1.15 — matches `.site-name` scale.
- Section headings inside the markdown body (`<h2>`) rendered at 28px / weight 700 / line-height 1.2 — inherits `.post-body h2` rule.
- Sub-headings (`<h3>`) rendered at 18px / weight 700 / line-height 1.2 — same size as body text, distinguished by weight alone.

**Source:** `body`, `.site-name`, `.post-body h2`, `.post-body h3` rules in `web/static/main.css`.

## Color

### Light mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F0E8` | Page background (`--color-bg`) |
| Secondary (30%) | `#E8DFD0` | Nav bar, cards, TOC, code blocks (`--color-surface`) |
| Accent (10%) | `#2C5F7A` | Reserved-for list below |
| Destructive | none | No destructive actions in this phase |

### Dark mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1A1F2E` | Page background |
| Secondary (30%) | `#242B3D` | Nav bar, cards, code blocks |
| Accent (10%) | `#C9A84C` | Reserved-for list below (same semantic role, warm gold) |

### Accent reserved for:
1. About nav link — hover state color only (`color: var(--color-accent)`)
2. Focus ring on the About nav link (`outline: 2px solid var(--color-accent)`)
3. Links inside the about markdown body (`.post-body a` — color + underline)
4. Left border on code fences inside the about markdown body (`.post-body pre` border-left)
5. Inline code background — via `--color-surface` (not accent directly)

Accent is NOT used for: the About nav link default state, body text, headings, or any decorative element.

**Source:** `:root` and `[data-theme="dark"]` custom properties in `web/static/main.css`.

### Nav link spec (`.nav-link`)

The "About" link sits between `.site-tagline` and `.dark-toggle` in the flex row. Per decision D-01 and D-02:

```css
.nav-link {
  font-size: 14px;              /* matches .site-tagline */
  color: var(--color-text-muted);
  text-decoration: none;
  margin-left: 16px;            /* 1 × md spacing from tagline */
  min-height: 44px;             /* touch target */
  display: inline-flex;
  align-items: center;
  white-space: nowrap;
}

.nav-link:hover {
  color: var(--color-accent);
}

.nav-link:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
}
```

On mobile (`max-width: 767px`): the tagline already hides. The About link must remain visible. Use `display: inline-flex` (not tied to tagline visibility).

**Source:** Decision D-01, D-02 from `08-CONTEXT.md`; `.site-tagline`, `.dark-toggle` patterns from `web/static/main.css`.

## Page Structure

**Primary focal point:** `<h1 class="about-title">` — 36px Playfair Display, first element in the content area.

The about page template follows the per-page template set pattern from `post.html`:

```
base.html chrome
  └── <nav class="site-nav">
        ├── .site-name (link to /)
        ├── .site-tagline (hidden mobile)
        ├── .nav-link "About" ← NEW, between tagline and toggle
        └── .dark-toggle
  └── {{define "content"}}
        <div class="container container-narrow">
          <article class="about-page">
            <h1 class="about-title">About</h1>
            <hr class="rope-divider">
            <div class="post-body">{{.RenderedHTML}}</div>
          </article>
        </div>
  └── <footer class="site-footer"> (inherited from base)
```

No reaction bar, no TOC, no post metadata line — the about page is not a blog post.

|---------|------|

## Markdown Content Outline (`content/about.md`)

Per decision D-05, Claude drafts initial content. The markdown file should follow this structure:

```markdown
# About

[Opening paragraph: who Jared Wallace is — software engineer, brief professional identity]

## The Wild Meridian

[Why this blog exists — the purpose, the "dispatches from the deep end" spirit, what kind of writing lives here]

## What You'll Find Here

[Brief description of content: technical writing, observations, occasional deep-dives]

---

*Thanks for reading.*
```

Tone: nautical-adjacent without being cosplay. "Dispatches from the deep end" as established in the tagline sets the register — measured, slightly literary, not pirate-y.

**Source:** D-03, D-04, D-05 from `08-CONTEXT.md`.

No JavaScript required for the about page itself. Dark mode toggle is inherited from `base.html`.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| About nav link keyboard reachable | Rendered as `<a href="/about">` — naturally keyboard focusable |
| About nav link focus indicator | `outline: 2px solid var(--color-accent); outline-offset: 2px` (matches site-wide `:focus-visible` pattern) |
| About nav link touch target | `min-height: 44px` on `.nav-link` |
| Page heading hierarchy | `<h1>` for page title, `<h2>` for markdown sections — no heading level skips |
| Article landmark | About content wrapped in `<article class="about-page">` |
| Image alt text in about.md | Any images in markdown must include descriptive alt text |
