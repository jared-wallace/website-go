    **Dashboard handler** in `dashboard.go`:

    `Dashboard(w, r)`:
    1. Read filter from query: `filter := r.URL.Query().Get("filter")` -- valid values: `""` (all), `"published"`, `"drafts"`, `"deleted"`
    2. Call `h.svc.ListAll(r.Context())` to get all posts
    3. Filter in-memory based on the tab:
       - `""` or `"all"`: exclude deleted (where DeletedAt == nil)
       - `"published"`: DeletedAt == nil AND Published == true
       - `"drafts"`: DeletedAt == nil AND Published == false
       - `"deleted"`: DeletedAt != nil
    4. Read flash messages: `flash_success := h.sessions.PopString(r.Context(), "flash_success")` and `flash_error := h.sessions.PopString(r.Context(), "flash_error")` (per Pitfall 8 -- use PopString not GetString)
    5. Render "admin-dashboard.html" with data:
       ```go
       data := map[string]interface{}{
           "Posts":        filteredPosts,
           "Filter":       filter,
           "FlashSuccess": flash_success,
           "FlashError":   flash_error,
       }
       ```

    **Action handlers** in `actions.go`:

    All action handlers follow the same pattern: parse `{id}` from path, call service method, set flash message, redirect to dashboard.

    `DeletePost(w, r)`:
    1. Parse ID: `id, err := strconv.ParseInt(r.PathValue("id"), 10, 64)`
    2. Call `h.svc.SoftDelete(r.Context(), id)`
    3. Set flash: `h.sessions.Put(r.Context(), "flash_success", "Post deleted. Restore it from the Deleted tab.")` (per UI-SPEC copywriting)
    4. Redirect 303 to `/admin/posts?filter=deleted`

    `RestorePost(w, r)`:
    1. Parse ID from path
    2. Call `h.svc.Restore(r.Context(), id)`
    3. Flash: `"Post restored to drafts."`
    4. Redirect to `/admin/posts?filter=drafts`

    `PublishPost(w, r)`:
    1. Parse ID from path
    2. Call `h.svc.Publish(r.Context(), id)`
    3. Flash: `"Post published."`
    4. Redirect to `/admin/posts`

    `UnpublishPost(w, r)`:
    1. Parse ID from path
    2. Call `h.svc.Unpublish(r.Context(), id)`
    3. Flash: `"Post unpublished."`
    4. Redirect to `/admin/posts`

    Error handling for all: if service call fails, set `flash_error` to "Something went wrong. Try again." and redirect to `/admin/posts`.

    **Remove stubs** from `handler.go`: Replace the 501 stub implementations for Dashboard, DeletePost, RestorePost, PublishPost, UnpublishPost with the real implementations in the new files. Keep NewPost, EditPost, SavePost, Preview stubs for Plan 04.
