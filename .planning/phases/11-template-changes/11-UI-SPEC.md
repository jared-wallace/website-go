## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tag pill gaps, inline padding, toc item margins |
| sm | 8px | Compact element spacing, tag pill padding, nav link margin-left |
| md | 16px | Default element spacing, section padding inside cards, footer margin |
| lg | 24px | Section padding, card padding, footer padding, rope-divider margin |
| xl | 32px | Layout gaps, post body heading margins |
| 2xl | 48px | Major section breaks, empty-state padding, pagination margin-top |
| 3xl | 64px | Page-level container top/bottom padding |

Exceptions:
- Touch targets: minimum 44px height/width on all interactive elements (nav links, dark toggle, rss link, pagination links, toc toggle). Source: existing `.nav-link`, `.dark-toggle`, `.rss-link` rules in `main.css`.

**Phase 11 spacing specifics:**

| Element | Spacing Rule |
|---------|-------------|
| Footer two-section layout | `padding: 24px` total; inner sections use `gap: 24px` between left and right columns |
| Footer stacking breakpoint | Stacks vertically at `max-width: 767px` (matches existing card-grid breakpoint) |
| Hero section (list page) | `margin-bottom: 32px` separating hero from card-grid; hero heading `margin: 0 0 8px`; tagline `margin: 0 0 0` |
| Rope divider margin | `margin: 24px 0` — inherit existing rule |

## Color

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `#F5F0E8` (`--color-bg`) | `#1A1F2E` | Page background |
| Secondary (30%) | `#E8DFD0` (`--color-surface`) | `#242B3D` | Nav bar, post cards, TOC, footer sections |
| Accent (10%) | `#2C5F7A` (`--color-accent`) | `#C9A84C` | See reserved list below |
| Text primary | `#2C2418` (`--color-text`) | `#E8DFD0` | Body text, headings |
| Text muted | `#7A6A55` (`--color-text-muted`) | `#9A8B75` | Nav links, metadata, footer copy, taglines |
| Divider | `#C8B89A` (`--color-divider`) | `#3A4558` | Borders, rope SVG stroke, divider lines |
| Destructive | not applicable | not applicable | No destructive actions in this phase |

Source: `main.css` `:root` and `[data-theme="dark"]` blocks.

**Accent reserved for:**
- Site name hover state
- Nav link hover state
- Footer nav link hover state
- Dark mode toggle hover state
- RSS link hover state
- Focus-visible outline on all interactive elements (2px solid)
- `.cta-button` background fill
- Rope SVG stroke color via `var(--color-divider)` — NOTE: rope uses `--color-divider`, not accent (D-05)
- Tag pill semi-transparent fill base color (8% opacity in light, 12% in dark)
- Reaction button border and filled state

**Rope divider color contract:**
The SVG rope pattern uses `stroke="var(--color-divider)"` — not accent. This ensures it recedes visually rather than competing with content. Source: D-05.

## Interaction Contract

### Footer Layout Behavior

| State | Behavior |
|-------|---------|
| Desktop (≥768px) | Two columns side by side: nav section left, copyright/utility right. `display: flex; justify-content: space-between; align-items: center` |
| Mobile (<768px) | Stacks vertically, centered. `flex-direction: column; align-items: center; gap: 16px` |

Source: D-01, breakpoint matches existing `@media (max-width: 767px)` pattern in `main.css`.

### Footer Nav Links

| State | Style |
|-------|-------|
| Default | `color: var(--color-text-muted)`, `font-size: 14px`, `min-height: 44px` |
| Hover | `color: var(--color-accent)` |
| Focus-visible | `outline: 2px solid var(--color-accent); outline-offset: 2px` |

Reuse existing `.nav-link` class — do not create a separate class.

### Rope SVG Divider

| Property | Value |
|----------|-------|
| Pattern | Twisted two-strand rope (classic nautical dock line). Source: D-04 |
| Dimensions | `width: 100%`, `height: 12px` — thin, decorative |
| Stroke color | `var(--color-divider)` — adapts to light/dark automatically. Source: D-05 |
| Stroke width | `2px` |
| Fill | `none` |
| Scope | Footer `.rope-divider` only — CSS class updated so future uses inherit. Source: D-06 |
| `aria-hidden` | `true` — decorative SVG |
| `role` | not set — decorative |
| Existing `<hr>` | Replaced with `<svg class="rope-divider">` inline element |

### Hero Section (List Page)

| Property | Value |
|----------|-------|
| Placement | Inside `.container`, immediately before `.card-grid` or `.empty-state` |
| Rendered when | Always — hero is unconditional, not gated on `.Posts` |
| h1 tag | Required — this is the page's primary heading |
| Tagline tag | `<p class="list-hero-tagline">` |

Source: D-07, D-08.

### Nav Restructuring

| Change | Rule |
|--------|------|
| About link | Removed from `<nav class="site-nav">` — not hidden, fully removed |
| Dark toggle | Stays in primary nav, retains `margin-left: auto` |
| Primary nav aria-label | `aria-label="Main navigation"` added to `<nav class="site-nav">` |
| Footer nav element | New `<nav aria-label="Footer navigation">` wrapping About link + RSS link |

Source: D-09, D-10.

## Accessibility Contract

| Requirement | Rule |
|-------------|------|
| Landmark disambiguation | Two `<nav>` elements on every page — each MUST have a distinct `aria-label` |
| Primary nav label | `aria-label="Main navigation"` |
| Footer nav label | `aria-label="Footer navigation"` |
| Touch targets | All footer links maintain `min-height: 44px` via `.nav-link` class |
| Rope SVG | `aria-hidden="true"` — purely decorative |
| Hero h1 | Must be the only `<h1>` on the list page — post card titles use `<h2>` |
| Dark mode transitions | Already gated behind `.theme-ready` (Phase 10) — no regression |
| Reduced motion | No new animations introduced; existing guards apply |

Source: NAV-04, WCAG 2.1 landmark labeling requirements.
