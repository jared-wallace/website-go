## Motion Contract

This is the primary contract for Phase 10. All values sourced from CONTEXT.md decisions.

### Animation Philosophy

"Whisper" — animations make the site feel polished without drawing attention. A visitor thinks
"this site feels nice," not "look at those animations."

### Timing Baseline

| Name | Value | Precedent |
|------|-------|-----------|
| Base transition | 200ms | Existing `.post-card`, `.reaction-btn` transitions |
| Fade-in duration | 250ms | D-01: slightly above baseline for entry feel |
| Dark mode blend | 250ms | D-05: matches fade-in for consistency |
| Stagger interval | 75ms | D-03: fast cascade between cards |
| Stagger cap | 6 cards | D-03: cards 7+ appear with no delay |
| Total cascade | ~375ms | D-04: 6 × 75ms |

### Easing Functions

| Context | Function | Rationale |
|---------|----------|-----------|
| Page fade-in | `ease-out` | Snappy entry, slows at end — natural feel |
| Card stagger | `ease-out` | Consistent with page fade-in (same keyframe reuse candidate) |
| Dark mode blend | `ease` | Symmetric color transition — no directional implication |
| Card hover lift | `ease` (existing) | Already established — do not change |

### Keyframes

**`@keyframes fade-in`** — opacity 0 → 1, no translate, no slide (D-01).
Card stagger SHOULD reuse this keyframe rather than defining a separate one (Claude's discretion
per CONTEXT.md, but DRY principle favors reuse given identical visual behavior).

**`@keyframes reaction-bounce`** — already defined at main.css line 653. Wrap in
`prefers-reduced-motion` guard only; do not change keyframe values.

### Selectors Receiving New Animation Rules

| Selector | Animation | Notes |
|----------|-----------|-------|
| `.main-content` | `fade-in 250ms ease-out` on page load | Navigation and chrome appear instantly — nav excluded |
| `.post-card:nth-child(1)` through `:nth-child(6)` | `fade-in 250ms ease-out` + `animation-delay: 0ms, 75ms, 150ms, 225ms, 300ms, 375ms` | Cards 7+ get no delay |
| `html, body, .site-nav, .post-card, .site-footer` (and other themed surfaces) | `transition: background-color 250ms ease, color 250ms ease, border-color 250ms ease` | Scoped behind `.theme-ready` selector |

Dark mode transition selector list (Claude's discretion per CONTEXT.md — recommended minimum):
`html`, `body`, `.site-nav`, `.post-card`, `.site-footer`, `.dark-toggle`, `.nav-link`,
`.reaction-btn`, `.tag-pill`. Executor should expand if visual testing reveals missed surfaces.

### Flash Prevention / `.theme-ready` Gate

**Inline script placement:** Before `<link rel="stylesheet">` in `<head>` (D-07).

**Script behavior (D-07, D-08):**
1. Read `localStorage('theme')` — set `data-theme="dark"` on `<html>` if indicated.
2. Check `window.matchMedia('(prefers-color-scheme: dark)')` as fallback.
3. After first paint: call `requestAnimationFrame(() => document.documentElement.classList.add('theme-ready'))`.

**Consolidation opportunity (Claude's discretion per CONTEXT.md):** The existing inline script
in `base.html` line 7 already handles steps 1–2. The executor SHOULD extend that script to add
the `requestAnimationFrame` call for `.theme-ready` rather than inserting a second inline script.
This keeps `<head>` clean.

**CSS gate:** All `transition` rules for dark mode surfaces are scoped as:
```
.theme-ready .site-nav,
.theme-ready .post-card,
/* etc. */
{ transition: background-color 250ms ease, color 250ms ease, border-color 250ms ease; }
```
Or equivalently: `.theme-ready *` with appropriate specificity — executor's choice based on
specificity conflict risk with existing transitions.

### Reduced-Motion Guard

All new animations AND the existing `reaction-bounce` are disabled under:

```css
@media (prefers-reduced-motion: reduce) {
  /* disable all animations and transitions */
}
```

**Policy (D-10, D-11):** Zero animation for reduced-motion users. No shortened/degraded
compromise. Instant state changes only. Guard covers:
- `@keyframes fade-in` (new)
- `.main-content` animation
- `.post-card:nth-child(n)` stagger animations
- `.theme-ready *` / surface transitions
- `reaction-bounce` (existing — currently unguarded, line 649–658 main.css)
